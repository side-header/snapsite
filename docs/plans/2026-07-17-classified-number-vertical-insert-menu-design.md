# 공종 번호 고정 및 세로 삽입 메뉴 설계

## 목표

공종 번호를 각 공종 영역의 실제 가장 왼쪽 상단에 고정하고, 제목을 바로 오른쪽에 배치한다. 번호 호버 시 페이지 삽입 버튼은 번호의 오른쪽 아래에 세로로 표시한다.

## 원인

번호 호스트가 번호 열과 제목 열을 함께 차지하도록 두 열에 걸쳐 배치되어, 고정 폭 번호가 넓은 영역의 가운데에 표시됐다.

## 기본 배치

- 번호 호스트를 헤더 열에서 분리해 공종 전체 레이어의 좌측 상단 `(0,0)`에 배치한다.
- 번호 호스트는 명시적으로 좌측·상단 정렬한다.
- 제목은 36px 번호 열 바로 오른쪽의 상단에 표시한다.
- 페이지당 사진 수와 삭제 버튼은 기존 오른쪽 위치를 유지한다.

## 호버 메뉴

- 메뉴는 번호 오른쪽 아래에서 시작해 번호를 가리지 않는다.
- 버튼은 `↑ 빈 페이지`, `↓ 빈 페이지`, `↑ 공종 페이지`, `↓ 공종 페이지` 순서로 세로 배치한다.
- 메뉴는 공종 전체 레이어의 오버레이로 표시해 사진 영역 위로 내려와도 잘리거나 뒤에 가려지지 않게 한다.
- 번호에서 메뉴로 마우스를 이동해도 호버 영역이 유지되도록 호스트 크기를 메뉴 영역까지 확장한다.

## 검증

- 캡처 기준으로 번호가 제목보다 왼쪽에 표시되는지 확인한다.
- 제목이 번호 바로 오른쪽 상단에 표시되는지 확인한다.
- 네 버튼이 세로로 표시되고 번호를 덮지 않는지 확인한다.
- 메뉴의 모든 버튼을 클릭할 수 있고 기존 삽입 동작이 유지되는지 확인한다.
- 프로젝트 빌드와 실행 중인 앱 재시작을 확인한다.
