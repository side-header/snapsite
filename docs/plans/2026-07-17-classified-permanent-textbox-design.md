# 분류된 공종 제목과 파일 이름 영구 TextBox 설계

## 목표

분류된 영역의 공종 제목과 사진 파일 이름을 표시용 `TextBlock`과 편집용 `TextBox` 사이에서 교체하지 않고, 항상 같은 `TextBox`로 표시한다.

## 공종 제목

- `ContentControl`과 표시용 `TextBlock` 교체 구조를 제거한다.
- 하나의 `TextBox`를 항상 사용한다.
- 왼쪽 정렬, 높이 28px, 글자 크기 13px, 줄 높이 18px를 유지한다.
- 포커스가 없을 때는 입력선이 보이지 않고, 클릭하면 같은 위치에 입력선만 나타난다.
- Enter로 저장하고, 포커스를 잃으면 저장하지 않은 값을 기존 제목으로 복구한다.

## 분류된 사진 파일 이름

- 분류된 사진 셀 전용의 영구 `TextBox`를 사용한다.
- 가운데 정렬, 높이 24px와 한 줄 표시를 유지한다.
- Enter로 파일 이름을 변경하고, 포커스를 잃으면 저장하지 않은 값을 기존 파일 이름으로 복구한다.
- 앱 화면 생성 시 자동 포커스하지 않는다.
- 분류되지 않은 영역과 탐색기의 표시·편집 교체 구조는 유지한다.

## 공통 동작

- 기존 파란 호버 테두리를 유지한다.
- 사진 드래그, 미리보기와 공종 선택 동작을 유지한다.

## 검증

- 두 영역이 평상시와 편집 시 같은 `TextBox`와 같은 글자 위치를 사용하는지 확인한다.
- Enter 저장과 포커스 해제 시 복구를 확인한다.
- 첫 화면에서 파일 이름 입력창이 자동 포커스되지 않는지 확인한다.
- 다른 영역의 파일 이름 동작이 유지되는지 확인한다.
- 프로젝트 빌드와 앱 재실행을 확인한다.
