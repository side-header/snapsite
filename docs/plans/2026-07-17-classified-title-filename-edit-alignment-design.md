# 분류된 제목과 파일 이름 편집 위치 정렬 설계

## 목표

분류된 영역의 공종 제목과 사진 파일 이름이 표시 상태에서 편집 상태로 전환될 때 글자 위치가 움직이지 않도록 한다.

## 공종 제목

- 표시·편집 상태 모두 글자 크기를 13px로 통일한다.
- 공통 호스트 높이를 28px로 고정한다.
- 두 상태 모두 왼쪽 정렬과 동일한 세로 중앙 기준을 사용한다.
- 편집 입력창의 내부 왼쪽 패딩을 0으로 설정한다.

## 분류된 사진 파일 이름

- 표시·편집 상태 모두 높이를 24px로 고정한다.
- 두 상태 모두 한 줄 및 가운데 정렬을 사용한다.
- 표시 상태에서 긴 이름은 말줄임으로 표시한다.
- 편집 상태에서는 전체 파일 이름을 입력창에 표시한다.
- 분류되지 않은 영역과 탐색기의 파일 이름 레이아웃은 변경하지 않는다.

## 유지 동작

- 기존 호버 파란 테두리를 유지한다.
- 클릭 편집, Enter 저장과 포커스를 잃을 때 표시 상태로 복귀하는 동작을 유지한다.
- 이미 표시·편집 위치가 같은 셀 제목은 변경하지 않는다.

## 검증

- 공종 제목과 사진 파일 이름을 클릭할 때 글자 시작 위치와 기준선이 움직이지 않는지 확인한다.
- 긴 사진 파일 이름의 표시 말줄임과 편집 전체 이름을 확인한다.
- 기존 저장 및 포커스 복귀 동작을 확인한다.
- 프로젝트 빌드와 앱 재실행을 확인한다.
