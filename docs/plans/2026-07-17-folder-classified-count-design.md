# 폴더별 분류 사진 개수 표시 설계

## 목표

분류되지 않은 영역의 폴더 줄 오른쪽에 기존 선택 사진 개수와 함께 분류된 사진 개수를 표시한다.

## 표시 규칙

- 선택 사진과 분류 사진이 모두 있으면 `2개 선택됨 · 5개 분류됨`처럼 한 줄로 표시한다.
- 선택 또는 분류 개수가 0이면 해당 항목만 숨긴다.
- 두 개수가 모두 0이면 상태 문구 전체를 숨긴다.
- 기존과 동일한 회색 글씨와 위치를 사용한다.

## 집계 범위

- 각 폴더의 바로 아래에 있는 사진만 집계한다.
- 하위 폴더의 사진은 상위 폴더 개수에 합산하지 않는다.
- 현재 열린 기준 폴더에서 스캔된 사진 중 공종에 배치된 사진만 `분류됨`으로 계산한다.
- 기준 폴더 바로 아래 사진은 기존 루트 폴더 키에 집계한다.

## 갱신 흐름

사진 선택 상태를 갱신할 때 선택 사진과 분류 사진의 폴더별 개수를 함께 계산한다. 사진을 공종에 추가하거나 분류에서 제거한 뒤 화면이 새로 고쳐지면 같은 표시 함수가 최신 상태를 반영한다.

## 검증

- 선택만 있는 폴더, 분류만 있는 폴더, 둘 다 있는 폴더, 둘 다 없는 폴더의 문구를 확인한다.
- 상위 폴더와 하위 폴더의 사진 개수가 서로 합산되지 않는지 확인한다.
- 프로젝트 빌드가 성공하는지 확인한다.
