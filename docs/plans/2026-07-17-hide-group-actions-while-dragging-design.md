# 공종 번호 드래그 중 작업 버튼 숨김 설계

## 목적

분류된 영역에서 공종 번호를 드래그하는 동안 공종 번호 옆의 페이지 추가 버튼과 파란 테두리가 계속 표시되는 문제를 해결한다.

## 동작

- 포인터 이동 거리가 드래그 임계값을 넘어 실제 공종 드래그가 시작되는 순간 버튼 팝업과 공종 번호의 파란 테두리를 숨긴다.
- 드롭하거나 드래그를 취소한 직후에도 현재 포인터가 공종 번호 영역 위에 있다는 이유만으로 팝업을 다시 표시하지 않는다.
- 포인터가 공종 번호 영역을 완전히 벗어난 뒤 다시 들어오면 기존 hover 동작을 복원한다.
- 클릭만 하고 드래그하지 않은 경우에는 기존 팝업과 공종 사진 탐색 동작을 유지한다.

## 구현 범위

공종 번호 컨트롤이 드래그 후 재진입 대기 상태를 관리한다. 공종 행의 드래그 처리기는 실제 드래그를 시작할 때 해당 컨트롤에 숨김을 요청한다. 창 전체 상태나 다른 공종 행은 변경하지 않는다.

## 검증

- 관련 테스트 프로젝트가 없으므로 전체 애플리케이션을 빌드한다.
- 앱을 재시작해 정상 실행 여부를 확인한다.
- 드래그 시작, 드래그 종료 후 정지, 포인터 이탈 후 재진입의 표시 규칙을 코드 경로로 확인한다.
