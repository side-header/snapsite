# 분류된 공종 인플레이스 삭제 설계

## 목적

분류된 영역에서 공종의 삭제 버튼을 눌렀을 때 삭제 위치 아래의 모든 공종과 사진 셀을 다시 만드는 지연을 제거한다.

## 동작

- 삭제한 공종 행 하나만 오른쪽 패널에서 제거한다.
- 기존 아래 공종 행과 사진 셀, 썸네일, 가로 스크롤 상태는 유지한다.
- 삭제 위치 아래의 공종 번호는 기존 번호 텍스트만 직접 변경한다.
- 아래 공종에 속한 분류되지 않은 영역의 회색 사진 카드도 공종 번호 글자만 변경한다.
- 삭제한 공종에 있던 사진 카드만 회색 분류 상태에서 선택 가능한 일반 상태로 교체한다.
- 현재 선택된 공종을 삭제하면 바로 아래 공종을 선택하고, 마지막 공종이었다면 바로 위 공종을 선택한다.
- 헤더와 상세 영역을 갱신한 뒤 자동 저장한다.

## UI 등록 정리

삭제한 공종의 행, 번호 텍스트, 강조 표시, 사진 스크롤, 사진 카드 등록만 제거한다. 유지되는 공종의 등록과 컨트롤은 건드리지 않는다.

## 안전장치

오른쪽 패널의 행 개수나 공종별 컨트롤 등록이 예상과 다르면 기존 범위 부분 갱신으로 대체한다. 이 경우에도 전체 화면 갱신은 사용하지 않는다.

## 범위

이번 최적화는 공종 행의 `×` 삭제 버튼에만 적용한다. 공종 순서 이동은 기존 동작을 유지한다.

## 검증

- 관련 테스트 프로젝트가 없으므로 전체 애플리케이션을 빌드한다.
- 앱을 재시작해 정상 실행 여부를 확인한다.
- 삭제 시 유지 공종 행이 교체되지 않고 번호와 삭제 공종의 사진 카드만 바뀌는지 코드 경로를 확인한다.
