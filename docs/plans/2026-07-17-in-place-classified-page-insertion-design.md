# 분류된 페이지 인플레이스 삽입 설계

## 목적

공종 번호 팝업의 위·아래 빈 페이지 또는 공종 페이지 추가 버튼을 눌렀을 때 삽입 위치 아래의 모든 공종을 다시 만드는 지연을 제거한다.

## 동작

- 새 페이지에 해당하는 공종 행 하나만 생성해 오른쪽 패널의 지정 위치에 삽입한다.
- 기존 공종 행과 사진 셀, 썸네일, 가로 스크롤 상태는 그대로 유지한다.
- 삽입 위치 아래의 공종 번호는 기존 번호 텍스트만 직접 변경한다.
- 분류되지 않은 영역의 회색 사진 카드에 표시되는 공종 번호도 카드 재생성 없이 번호 텍스트만 변경한다.
- 신규 공종 페이지는 기존처럼 선택하고 상세 영역을 갱신한다.
- 신규 페이지 위치로 이동하는 스크롤과 렌더링 이후 자동 저장은 유지한다.

## UI 등록

공종 ID별 공종 번호 텍스트와 사진 경로별 분류 공종 번호 텍스트를 등록한다. 전체 또는 부분 갱신으로 컨트롤이 교체될 때 해당 등록도 함께 정리하고 새 컨트롤을 다시 등록한다.

## 안전장치

오른쪽 패널의 행 개수나 등록 상태가 예상과 다르면 기존 범위 부분 갱신으로 대체한다. 이 경우에도 전체 화면 갱신은 사용하지 않는다.

## 범위

이번 최적화는 공종 번호 팝업의 `위/아래 빈 페이지`, `위/아래 공종 페이지` 네 버튼에만 적용한다. 공종 삭제와 순서 이동은 기존 동작을 유지한다.

## 검증

- 관련 테스트 프로젝트가 없으므로 전체 애플리케이션을 빌드한다.
- 앱을 재시작해 정상 실행 여부를 확인한다.
- 삽입 시 기존 공종 행이 교체되지 않고 번호와 사진 카드 배지만 바뀌는지 코드 경로를 확인한다.
