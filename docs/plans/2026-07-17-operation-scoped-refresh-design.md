# 작업 범위 기반 부분 갱신 설계

## 목적

사용자 작업과 관계없는 화면까지 다시 그리는 `RefreshAll()` 호출을 제거해 화면 깜빡임과 입력 지연을 줄인다. 앱 초기화, 폴더 열기, 수동 재스캔, 썸네일 캐시 삭제처럼 전체 상태가 바뀌는 작업만 전체 갱신을 유지한다.

## 갱신 원칙

- 공종 추가, 삭제, 이동은 실제로 바뀐 공종 행과 번호가 달라진 구간만 갱신한다.
- 공종 번호가 변경되면 해당 공종에 연결된 분류되지 않은 사진 카드의 번호 표시도 함께 갱신한다.
- 사진 분류와 분류 해제는 관련 공종, 사진 카드, 폴더별 개수만 갱신한다.
- 한 페이지당 사진 수 변경은 해당 공종 하나만 갱신한다.
- 규칙1 또는 선택 사진으로 공종을 여러 개 만들 때는 새 공종과 선택된 사진 카드만 갱신한다.
- 기존 세로·가로 스크롤과 폴더 펼침 상태를 보존한다.

## 이름 변경

파일 또는 폴더 이름 변경이 성공하면 경로와 폴더 구조가 바뀌므로 왼쪽 트리와 분류되지 않은 영역을 갱신한다. 분류된 영역은 변경 경로를 포함하는 공종만 갱신한다. 파일 없음이나 이름 중복 같은 검증 실패는 상태 메시지만 표시하고 화면을 다시 그리지 않는다.

예외가 발생하면 실제 파일 시스템 변경 여부를 확인한다. 경로가 변경되지 않았다면 재렌더링하지 않고, 이동이 일부 완료된 경우에만 관련 트리, 사진 목록, 공종을 복구 갱신한다.

## 안전장치

부분 갱신에 필요한 컨트롤을 찾을 수 없거나 화면 구조가 예상과 다르면 관련 영역 단위 갱신으로 대체한다. 이 경우에도 `RefreshAll()`로 후퇴하지 않는다.

## 전체 갱신 허용 범위

`RefreshAll()`은 다음 작업에만 남긴다.

- 앱 초기화
- 폴더 열기
- 수동 저장에 수반되는 재스캔
- 썸네일 캐시 삭제

## 검증

- `RefreshAll()` 호출부가 허용된 네 범주에만 남았는지 검색한다.
- 부분 갱신 후 선택 상태, 공종 번호, 폴더별 개수, 스크롤 위치가 유지되는지 코드 경로를 확인한다.
- 테스트 프로젝트가 없으므로 전체 솔루션을 빌드한다.
- 앱을 재시작하고 프로세스가 정상 실행되는지 확인한다.
