# 폴더 아이콘 기반 규칙1 선택 디자인

## 목표

분류되지 않은 영역의 폴더 아이콘을 통해 해당 폴더와 모든 하위 폴더의 규칙1 사진을 선택하고 기존 상단 규칙1 버튼을 제거한다.

## 폴더 아이콘 동작

- 분류되지 않은 영역의 폴더 아이콘은 기본 상태에서 현재 밝은 황토색을 유지한다.
- 폴더 아이콘에 마우스를 올리면 기존 파란색 계열로 변경한다.
- 마우스가 아이콘을 벗어나면 밝은 황토색으로 복원한다.
- 폴더 아이콘을 클릭하면 해당 폴더와 모든 하위 폴더를 범위로 규칙1 선택을 실행한다.
- 폴더 아이콘 클릭은 행의 펼치기/접기 동작으로 전달하지 않는다.
- 화살표, 폴더명과 나머지 행을 클릭하면 기존처럼 폴더를 펼치거나 접는다.

## 규칙1 선택

- 선택한 폴더 경로와 같거나 그 경로 아래에 있는 모든 사진을 후보로 사용한다.
- 기존 규칙1 파일명 규칙과 공종 계획 생성 방식은 유지한다.
- 이미 분류된 사진은 제외한다.
- 선택 결과가 있으면 기존 하단 안내 상자와 공종 추가 버튼을 표시한다.
- 선택 결과가 없으면 기존 상태표시줄 메시지를 표시한다.
- 트리의 펼침 상태와 스크롤 위치는 변경하지 않는다.

## 상단 버튼

- 분류되지 않은 영역 우측 상단의 `규칙1 기준 선택` 버튼을 제거한다.
- 사진 확대·축소 버튼은 기존 위치와 동작을 유지한다.

## 검증

- 폴더 자체와 하위 폴더 포함 범위 계산을 특성 테스트로 검증한다.
- 형제 폴더의 경로가 포함되지 않는지 검증한다.
- 솔루션 전체 빌드와 기존 특성 테스트를 실행한다.
- 앱을 재실행해 변경된 UI가 실행되는 상태로 둔다.
